# goxlr-router

Desktop UI (Tauri + Vue) to inspect PipeWire nodes, ports, and links in a switchboard-style graph, and to create or remove links with `pw-link`. ALSA playback/capture lists (`aplay -l`, `arecord -l`) help correlate with hardware.

The graph is built in Rust from `pw-dump` and sent to the UI as a small JSON blob (full dumps are too large for Tauri IPC and were getting truncated, which looked like an empty graph).

## Requirements

- A **user graphical session** with PipeWire (same `XDG_RUNTIME_DIR` as your compositor).
- **`pw-dump`**, **`pw-link`**, **`aplay`**, and **`arecord`** on `PATH` when running the binary without the Nix wrapper.
- **GStreamer** plugins for WebKitGTK (the Nix package sets `GST_PLUGIN_SYSTEM_PATH_1_0`). If you run a locally built binary and see *GStreamer element appsink not found*, point that variable at `gst-plugins-base` (and usually `gst-plugins-good`), e.g. under NixOS install those packages and match the `default.nix` wrapper pattern.

## Local development

### 1. Toolchain (Nix example)

Open a shell with Node, Rust, GTK/WebKit build deps, and PipeWire/ALSA CLI tools:

```bash
nix-shell -p nodejs_22 cargo rustc pkg-config openssl gtk3 webkitgtk_4_1 libsoup_3 \
  pipewire alsa-utils
```

Install the **Tauri 2 CLI** once (pick one):

```bash
cargo install tauri-cli --locked
```

If your `nixpkgs` ships a `cargo-tauri` (or similar) package, you can use that instead of `cargo install`.

### 2. JavaScript dependencies

```bash
cd pkgs/goxlr-router
npm install
```

### 3. Run the app in dev mode

`tauri.conf.json` runs `npm run dev` (Vite on port 1420) before opening the window.

```bash
cd pkgs/goxlr-router/src-tauri
cargo tauri dev
```

- **Frontend only** (no native shell / no `invoke`): from `pkgs/goxlr-router`, run `npm run dev` and open the printed URL (IPC calls will not work without Tauri).

### 4. Typecheck and production frontend build

```bash
cd pkgs/goxlr-router
npm run build
```

### 5. Rust release binary (without Nix packaging)

After `npm run build`:

```bash
cd pkgs/goxlr-router/src-tauri
cargo build --release
# Binary: target/release/goxlr-router
```

Ensure `pw-dump`, `pw-link`, `aplay`, and `arecord` are on `PATH` when running the binary.

**Important:** use `cargo tauri build --no-bundle` (or the Nix package, which does this), not plain `cargo build --release` alone. A bare `cargo build` can produce a binary that still points the WebView at the Vite dev server (`http://localhost:1420`), which fails at runtime with **Connection refused**.

## Build with the flake (this repo)

From the flake root (`etc/nixos`):

```bash
nix build .#packages.x86_64-linux.goxlr-router
```

The Nix package wraps the binary so **pipewire** and **alsa-utils** are on `PATH`.

## Updating the npm lockfile / Nix hash

If you change `package.json`:

```bash
cd pkgs/goxlr-router
npm install
nix shell nixpkgs#prefetch-npm-deps -c prefetch-npm-deps package-lock.json
```

Copy the printed SRI hash into `npmDeps.hash` in `default.nix`.

## Updating Rust dependencies

After editing `src-tauri/Cargo.toml`:

```bash
cd pkgs/goxlr-router/src-tauri
cargo update
```

Commit the updated `Cargo.lock`. Nix uses `cargoLock.lockFile` in `default.nix`.

## Ignored paths

See `.gitignore`: `node_modules/`, `dist/`, `src-tauri/target/`, `src-tauri/gen/` (generated by Tauri during build).
